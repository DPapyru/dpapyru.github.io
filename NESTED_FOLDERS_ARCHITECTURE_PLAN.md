
# 多级嵌套文件夹目录结构架构方案

## 1. 现有系统分析

### 1.1 优势
- **已部分支持递归扫描**：[`generate-index.js`](generate-index.js:9) 中的 `scanDirectoryRecursively` 函数可以递归扫描子目录
- **路径字段已实现**：[`config.json`](docs/config.json:13) 中的文件对象同时包含 `filename`（仅文件名）和 `path`（完整相对路径）字段
- **前端路径处理**：[`viewer.html`](docs/viewer.html:603) 中的 `loadMarkdownDirectly` 函数已经有路径处理逻辑
- **已有子文件夹示例**：存在 `docs/给贡献者阅读的文章/` 子文件夹

### 1.2 局限性
- **目录结构不够清晰**：目前只有单一子文件夹示例，缺乏系统性组织
- **路径处理逻辑复杂**：前端路径处理代码较为复杂，需要简化
- **缺乏统一命名规范**：文件夹和文件命名不够规范
- **分类与目录结构不匹配**：现有的分类系统（入门、进阶等）与物理目录结构不对应
